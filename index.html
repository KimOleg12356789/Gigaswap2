OTC SWAP SMART CONTRACT SYSTEM
OPERATING PRINCIPLE
The main contract is the Swapper contract. This contract provides work with deals, which are called Deal.

Deal structure:
struct Deal {
uint256 state; // 0 - not exists, 1-editing 2-execution 3-swaped address
owner0; // owner 0 - creator
address owner1; // owner 1 - second part
}
Each deal is available by its ID, which is greater than 0. There is also a state in the deal, while:
  • 0 - if there is no such deal
  • 1 - the deal has been created and at the moment it is possible to edit its conditions
  • 2 - the deal cannot be edited (you can change the state only once in this state and only by any of the participants in the deal) at the moment, each deal is calculated on 2 sides, and the 0 side is always the creator.
Each deal consists of deals items described by the structure:
struct DealPointData {
address factory;
address point;
}
where the factory is the factory that generated the deal point and point is the address of the deal point contract. Draw the attention that each point of each deal is a separate smart contract with its own address, generated by a certain factory.
WHAT ARE DEAL FACTORIES FOR
OPERATING PRINCIPLE
Deal factories are needed in order to generate varied deal points. For example, there is such a factory (additional code omitted):
contract Erc20CountPointFactory is PointFactory { ...
function makePoint(
uint256 dealId,
address token,
uint256 needCount,
address from,
address to
) public;
...
}
This factory generates deal points that say that a certain address should transfer to another address the specified amount of the specified erc20 tokens. Also, the deal ID is passed to the generation function, which is obtained after the creation of the deal itself in the swapper. The factory is always paired with generated by the contract, for example, the factory described above generates a contract (extra code is omitted):
contract Erc20CountPoint is DealPoint {
IERC20 public token; uint256
public needCount;
address public from;
address public to;

function isComplete() external view override returns (bool); function withdraw() external;
}
as we see, this is a smart contract that knows how many of the specified tokens are on it and can report that it has been completed. There is also a command to withdraw tokens by the current owner. The current owner changes only if all the points in the deal are completed (isComplete). Note that these tokens can be transferred not only by the person specified in the deal point but also by any other address. If only the point of the transaction was completed, and the indicated address from whom the transfer of the asset is coming is purely for withdrawal, if you suddenly want to cancel the point of the deal and withdraw everything that lies on it.
FACTORIES CURRENTLY AVAILABLE
OPERATING PRINCIPLE
  • Erc20CountPointFactory - generates deal points when there is a need to transfer the erc20 token in the specified amount
  • Erc721CountFactory - transfer of nft in the specified amount
  • Erc721ItemFactory - transfer of one specific nft
